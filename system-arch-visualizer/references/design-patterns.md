# 構成図設計のリファレンス

システム構成図を設計・描画する際により高度な表現が必要な場合は、以下のパターンを参照すること。

## 階層構造のベストプラクティス

### 3層構造 (3-Tier)
1. **現場層 (L1/Field)**: PLC, センサー, アクチュエータ
2. **収集・制御層 (L2/Control)**: エッジPC, OPCサーバ, ゲートウェイ
3. **蓄積・分析層 (L3/Information)**: DB, クラウド, ダッシュボード

### セキュリティ境界
- ネットワーク分離（DMZ, 社内網, 現場網）が必要な場合は、`subgraph` をさらにネストして表現する。

## 描画テクニック

### Mermaid ノード形状の使い分け
- `[ ]`: 一般的なプロセス、機器
- `[[ ]]`: サブプロセス、別リポジトリの機能
- `[( )]`: データベース、ストレージ
- `{{ }}`: 条件分岐、判断
- `> ]`: 非同期メッセージ、イベント通知

### 通信プロトコルの記述例
- `-->|OPC UA|` : 産業用標準
- `-.->|HTTPS|` : 疎結合なWeb通信
- `==>|USB/Serial|` : 物理的な直結
